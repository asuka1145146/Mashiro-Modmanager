

## 主要功能

### 📦 Mod 管理
- **导入 Mod**：支持单个 Mod 导入，自动识别 Mod 信息
- **批量导入**：支持从狩技盒子或导出目录批量导入 Mod
- **导出功能**：支持导出为单个合并 Mod 或导出为多个独立 Mod
- **启用/禁用**：一键启用或禁用 Mod，自动处理文件操作
- **Mod 编辑**：编辑 Mod 的名称、分类、作者等信息

### 🏷️ 分类与标签
- **标签管理**：自定义标签分类，支持标签的添加、删除、重命名
- **标签排序**：通过拖拽或交换功能调整标签显示顺序
- **多标签支持**：每个 Mod 可设置多个标签，按优先级排序显示

### ⚡ 批量操作
- **批量禁用**：快速禁用所有 Mod 或使用二分法定位问题 Mod
- **批量导出**：选择多个 Mod 进行批量导出操作
- **智能合并**：检测冲突后，将多个无冲突的 Mod 合并为单个 Mod

### 🔍 冲突检测
- **自动检测**：启用 Mod 时自动检测文件冲突
- **冲突处理**：提供多种冲突解决方案（直接覆盖、手动调整优先级）
- **优先级管理**：通过可视化界面调整 Mod 加载优先级

### ⭐ 便捷功能
- **收藏功能**：标记常用 Mod，快速定位
- **搜索过滤**：快速搜索和筛选 Mod
- **统计信息**：实时显示 Mod 总数、启用数量等统计信息
- **使用记录**：记录 Mod 的启用/禁用历史

### 🎨 个性化设置
- **主题定制**：自定义界面颜色、透明度等主题设置
- **高级设置**：配置游戏路径、虚拟映射等高级选项
- **界面动画**：流畅的界面过渡动画，提升使用体验

## 系统要求

- **操作系统**：Windows 10 或更高版本
- **Python 环境**（开发版本）：Python 3.8 或更高版本
- **依赖库**：PySide6

## 安装与使用

### 使用已打包版本

1. 下载最新版本的 `ModManager.exe`
2. 双击运行即可使用

### 从源码运行

1. 确保已安装 Python 3.8 或更高版本
2. 安装依赖：
   ```bash
   pip install PySide6
   ```
3. 运行主程序：
   ```bash
   python main.pyw
   ```

## 使用指南

### 首次使用

1. **设置游戏路径**：点击"高级设置"，填写游戏安装目录路径
2. **导入 Mod**：点击"导入 Mod"按钮，选择 Mod 文件或文件夹
3. **启用 Mod**：勾选 Mod 前的复选框即可启用

### 基本操作

- **启用/禁用 Mod**：点击表格左侧的复选框
- **编辑 Mod 信息**：双击 Mod 行或右键选择编辑
- **搜索 Mod**：在搜索框中输入关键词
- **收藏 Mod**：右键点击 Mod 选择收藏

### 批量操作

- **批量禁用**：点击"批量禁用"按钮，选择禁用方式
- **批量导出**：选择多个 Mod 后，点击"导出选中"，选择导出方式
- **合并 Mod**：选择多个无冲突的 Mod，选择"导出为 mod"进行合并

### 标签管理

1. 点击"标签管理"按钮
2. 在标签管理界面中：
   - 添加标签：输入标签名称，点击"添加"
   - 删除标签：点击标签行的"删除"按钮
   - 重命名标签：点击标签行的"重命名"按钮
   - 调整顺序：按住 Ctrl 键选择两个标签，点击"交换"按钮

## 注意事项

1. **游戏路径设置**：首次使用前请务必在高级设置中配置正确的游戏路径
2. **Mod 冲突**：启用 Mod 时如检测到冲突，请根据提示选择合适的处理方式
3. **虚拟映射**：使用虚拟映射功能需要管理员权限，建议在启用前禁用所有 Mod
4. **备份重要数据**：建议定期备份 `json` 目录下的配置文件

## 常见问题

**Q: 为什么启用 Mod 后没有效果？**  
A: 请检查游戏路径是否正确设置，以及 Mod 文件是否完整。

**Q: 如何解决 Mod 冲突？**  
A: 可以选择直接覆盖（可能覆盖其他 Mod 的文件），或手动调整优先级。

**Q: 批量导出时如何合并 Mod？**  
A: 选择多个 Mod 后，选择"导出为 mod"，系统会自动检测冲突。若无冲突，将进入合并编辑界面。

**Q: 标签顺序如何调整？**  
A: 在标签管理界面，按住 Ctrl 键选择两个标签，点击"交换"按钮即可。
